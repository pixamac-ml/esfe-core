────────────────────────────────────────────────────────────────────────────
 CHANTIER A — Migration communication/ → notifier/ + notification_center/
────────────────────────────────────────────────────────────────────────────

C'est le plus gros changement. L'ancienne app communication/ mélangeait
deux choses très différentes dans le même dossier :
  1. Le MOTEUR D'ENVOI (EventBus, dispatchers, providers email, WebSocket)
  2. L'INTERFACE UTILISATEUR (centre de notifications, vues, templates)

Cela causait une confusion permanente. La décision a été de séparer les
deux en deux apps Django distinctes :

  notifier/              → LE MOTEUR D'ENVOI (infrastructure pure)
  ─────────────────────────────────────────────────────────────────
  - NotificationBus : API unique (notify, send_email, mark_as_read)
    remplace les 3 anciennes APIs : NotificationService, EmailService,
    CommunicationEventBus
  - Modèles renommés : NotificationEvent, NotificationMessage, DeliveryAttempt
    (ex CommunicationEvent, CommunicationNotification, CommunicationDelivery)
  - Dispatcher, channels, policy engine, audience resolver
  - Providers email (Brevo)
  - WebSocket consumer (ws/notifications/)
  - Migrations créées et validées
  - Tests unitaires (2 tests)

  notification_center/   → L'INTERFACE UTILISATEUR (lecture seule)
  ─────────────────────────────────────────────────────────────────
  - Vues : centre de notifications, widget cloche, dropdown, marquer lu/non-lu
  - URLs sous /notifications/
  - Selecteurs : get_user_messages, get_user_unread_count, get_stats...
  - Context processor global (remplace communication.context_processors)
  - Templates : index, list, item, detail, dropdown, widget (avec WebSocket JS)
  - Tests : test_views.py, test_selectors.py

Migration des call sites (85 appels dans 15+ apps) :
  - admissions, inscriptions, payments, shop, community
  - secretary, portal, marketing, academics, accounts
  - core, students, superadmin
  Tous migrés de NotificationService.notify_user() → NotificationBus.notify()

Config nettoyée :
  - config/urls.py : URLs communication/ retirées, notification_center/ ajoutée
  - config/asgi.py : WebSocket communication/ retiré, notifier/ ajouté
  - config/settings.py : context_processor mis à jour
  - ui/navbar : importe maintenant depuis notification_center (plus communication)

Statut de communication/ :
  - Toujours dans INSTALLED_APPS (nécessaire pour les migrations DB)
  - Plus aucun import externe
  - Plus aucune URL accessible
  - Prochaine étape : migration des données DB, puis suppression définitive
