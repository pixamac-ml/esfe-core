# Role de l'informaticien dans les notes et passages

L'informaticien est le responsable technique du systeme de calcul des notes.
Il ne decide pas du passage d'un etudiant. Le passage est calcule automatiquement par les regles academiques.

## Ce qu'il doit faire

1. Verifier la structure academique :
   - classes ;
   - semestres ;
   - UE ;
   - EC ;
   - coefficients ;
   - credits ;
   - seuil de validation.

2. Saisir ou importer les notes :
   - saisie manuelle ;
   - import Excel ;
   - controle des notes invalides ;
   - controle des notes manquantes.

3. Controler la completude :
   - etudiants sans notes ;
   - semestres incomplets ;
   - EC mal configures ;
   - credits incoherents.

4. Verifier les calculs :
   - moyenne EC ;
   - moyenne UE ;
   - moyenne semestre ;
   - moyenne annuelle ;
   - credits obtenus ;
   - rattrapages ;
   - dettes academiques.

5. Gerer les etapes techniques :
   - verrouiller la session normale ;
   - ouvrir les rattrapages ;
   - verrouiller les rattrapages ;
   - publier ou envoyer les resultats calcules au workflow de validation.

## Ce qu'il ne doit pas faire

- Il ne force pas le passage.
- Il ne modifie pas une decision calculee par les regles.
- Il ne valide pas pedagogiquement a la place du directeur des etudes.
- Il ne donne pas le visa financier.
- Il ne finalise pas seul la reinscription administrative.

## Dans son dashboard

Il doit voir :

- les classes avec notes incompletes ;
- les notes a corriger ;
- les semestres calculables ;
- les resultats calcules ;
- les cas de passage normal ;
- les cas de passage avec dette ;
- les redoublements ;
- les cas incomplets ;
- les anomalies bloquantes.

Les bons boutons pour lui :

- Recalculer ;
- 
- Voir detail ;
- Corriger notes ;
- Importer notes ;
- Ouvrir rattrapage ;
- Verrouiller session ;
- Envoyer au workflow de validation.

Il ne doit pas avoir de bouton "Forcer passage".
