A AJOUTER EN PRIORITE
=====================

1. Filtres de pilotage
----------------------
- Filtre par annexe.
- Filtre par periode : aujourd'hui, semaine, mois, annee scolaire.
- Filtre par domaine : academique, finance, inscriptions, surveillance.
- Bouton "Reinitialiser les filtres".

2. Detail annexe
----------------
- Une page ou un panneau detail pour chaque annexe.
- Voir les classes, les responsables, les effectifs, les paiements, les alertes.
- Comparer une annexe avec les autres.
- Afficher les tendances : hausse/baisse inscriptions, revenus, absences.

3. Actions depuis le dashboard
------------------------------
- Ouvrir une annexe.
- Voir les paiements de l'annexe.
- Voir les inscriptions de l'annexe.
- Voir les alertes de l'annexe.
- Ouvrir le pilotage des passages/reinscriptions.
- Exporter les donnees en Excel ou PDF.

4. Finance DG
-------------
- Ajouter un tableau des revenus par annexe.
- Ajouter les depenses par annexe.
- Ajouter le solde estime : revenus - depenses.
- Ajouter les paiements en attente ou rejetes.
- Ajouter un classement des annexes par performance financiere.

5. Academique DG
----------------
- Ajouter le taux de remplissage des classes.
- Ajouter les classes sans effectif ou trop chargees.
- Ajouter les semestres en retard.
- Ajouter les resultats non valides ou non publies.
- Ajouter les enseignants actifs par annexe.
- Ajouter les cours planifies / cours realises.

6. Alertes et risques
---------------------
- Mettre les alertes critiques en haut du dashboard.
- Categoriser les alertes :
  - finance,
  - absences,
  - resultats,
  - inscriptions,
  - systeme.
- Ajouter un statut : nouveau, en cours, resolu.
- Ajouter un responsable de traitement.

7. Workflow passages et reinscriptions
--------------------------------------
- Afficher clairement les etapes :
  1. Decision academique.
  2. Validation finance.
  3. Application du passage.
  4. Reinscription confirmee.
- Ajouter un compteur par etape.
- Ajouter une liste des dossiers bloques.
- Ajouter un bouton pour ouvrir chaque dossier.


A RETIRER OU LIMITER
====================

- Eviter les blocs trop longs par annexe sur la page principale.
- Ne pas afficher trop de lignes de paiements dans la vue globale.
- Ne pas melanger les indicateurs strategiques DG avec les actions operationnelles trop fines.
- Eviter les textes explicatifs trop longs dans les cartes.
- Garder les details dans des panneaux ou pages secondaires.


DESIGN A AMELIORER
==================

1. Hierarchie visuelle
----------------------
- Mettre 4 ou 5 KPIs principaux en haut.
- Mettre les alertes critiques juste apres les KPIs.
- Mettre les annexes dans un tableau comparatif plus compact.
- Garder les details dans un panneau lateral ou une page detail.

2. Tableau comparatif des annexes
---------------------------------
Colonnes proposees :
- Annexe
- Responsable
- Etudiants
- Classes
- Inscriptions
- Revenus
- Depenses
- Alertes
- Statut

3. Couleurs
-----------
- Vert : situation normale.
- Orange : attention.
- Rouge : critique.
- Bleu : information ou suivi.
- Gris : donnees neutres.

4. Navigation
-------------
- Menu gauche plus clair :
  - Vue globale
  - Annexes
  - Finance
  - Academique
  - Passages
  - Alertes
  - Exports

5. Ergonomie
------------
- Ajouter des boutons d'action courts.
- Ajouter des icones pour reconnaitre rapidement les modules.
- Garder les cartes petites et lisibles.
- Eviter une page trop longue.


WORKFLOW IDEAL DU DG
====================

1. Le DG se connecte.
2. Il arrive sur une vue globale claire.
3. Il voit d'abord :
   - total etudiants,
   - total revenus,
   - annexes actives,
   - alertes critiques,
   - dossiers de passage bloques.
4. Il filtre par periode ou par annexe si besoin.
5. Il clique sur une annexe pour voir le detail.
6. Il ouvre les alertes ou dossiers bloques.
7. Il exporte un rapport si necessaire.


PROCHAINE ETAPE CONSEILLEE
==========================

Transformer la section "Annexes" actuelle en tableau comparatif compact,
puis ajouter un panneau detail annexe. C'est le plus utile pour un DG,
car il doit comparer vite et agir seulement quand une annexe pose probleme.
