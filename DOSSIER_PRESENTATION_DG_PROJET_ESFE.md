## 1. Résumé exécutif

Le projet **ESFE** est une plateforme numérique complète conçue pour soutenir la visibilité, la gestion et la croissance de l’établissement.

Ce projet ne se limite pas à un simple site vitrine. Il s’agit d’un **écosystème digital unifié** qui permet à l’ESFE de :

- présenter son identité institutionnelle de manière professionnelle ;
- promouvoir ses formations et ses filières ;
- recevoir et traiter les candidatures en ligne ;
- transformer les candidatures acceptées en inscriptions officielles ;
- gérer les paiements et les justificatifs associés ;
- créer automatiquement les profils étudiants après validation des paiements ;
- publier des actualités, articles, événements et contenus institutionnels ;
- administrer l’ensemble depuis un cockpit central appelé **superadmin** ;
- permettre à des agents autorisés de travailler dans des dashboards dédiés à leurs missions quotidiennes.

En résumé, le projet ESFE positionne l’école comme une institution moderne, crédible, structurée et prête à se développer à l’échelle nationale et internationale.

## 2. Vision générale du projet

### 2.1. Objectif principal

Mettre à la disposition de l’ESFE une plateforme capable de répondre à trois grands enjeux :

1. **Image et notoriété** : rendre l’école plus visible, plus attractive et plus crédible.
2. **Performance opérationnelle** : fluidifier les processus internes liés aux admissions, inscriptions et paiements.
3. **Pilotage stratégique** : donner à la Direction Générale une vue claire sur les activités et les performances.

### 2.2. Positionnement institutionnel

Le projet valorise l’ESFE comme :

- un établissement supérieur orienté vers les sciences de la santé ;
- une structure organisée avec identité claire, annexes/campus, filières et cycles définis ;
- une institution capable d’offrir une expérience numérique cohérente au public, aux candidats, aux agents et à la direction.

### 2.3. Ce que le projet change concrètement

Avant un tel système, plusieurs opérations sont souvent dispersées, manuelles, lentes ou peu traçables.  
Avec ESFE, l’établissement dispose d’un environnement unique pour :

- centraliser les informations ;
- standardiser les processus ;
- suivre les statuts des dossiers ;
- réduire les pertes d’information ;
- sécuriser les opérations ;
- améliorer la qualité de service rendue aux apprenants et aux familles.

La plateforme est organisée autour de **deux grands univers complémentaires**.

### 3.1. Univers public

Elle comprend notamment :

- la page « À propos » ;
- les pages formations ;
- les pages de détail des formations ;
- les actualités ;
- le blog ;
- la communauté ;
- la page contact ;

C’est la partie réservée à l’administration, aux responsables et aux agents autorisés.

Elle comprend notamment :

- la gestion des inscriptions ;
- la gestion des paiements ;
- la gestion des étudiants ;
- la gestion des contenus institutionnels et marketing.

## 4. Présentation des grands modules métier

### 4.1. Module institutionnel

Ce module porte l’image officielle de l’école.

Il permet de présenter :

- le nom de l’institution ;
- les coordonnées ;
- le statut juridique ;
- l’agrément ;
- la direction ;
- l’hébergement du site ;
- les textes de présentation, mission, vision et messages institutionnels.

**Intérêt pour la DG :** ce module garantit une parole officielle, cohérente et centralisée.

---

### 4.2. Module formations

Ce module structure toute l’offre académique.

Il permet de gérer :

- les cycles ;
- les filières ;
- les diplômes ;
- les programmes ;
- les années de programme ;
- les frais par année ;
- les documents requis ;
- les contenus détaillés de chaque formation.

La page de détail d’une formation a une fonction stratégique : elle doit convaincre un visiteur de passer de la simple consultation à l’action.

Elle doit donc répondre à plusieurs questions clés :

- Pourquoi choisir cette formation ?
- Que va apprendre l’étudiant ?
- Quel diplôme sera obtenu ?
- Quels sont les débouchés ?
- Quel est le coût ?
- Comment candidater ?
- Quels anciens étudiants témoignent de la valeur du parcours ?

**Intérêt pour la DG :** c’est le cœur commercial et académique du site.

---

### 4.3. Module admissions

Le module admissions gère la phase de candidature.

Un candidat peut :

- choisir une formation ;
- choisir une annexe/campus ;
- soumettre ses informations personnelles ;
- déposer son dossier ;
- suivre l’évolution de sa candidature.

Côté établissement, le système permet de traiter les candidatures selon des statuts métier tels que :

- soumise ;
- en cours d’analyse ;
- à compléter ;
- acceptée ;
- acceptée sous réserve ;
- refusée.

**Intérêt pour la DG :** cela professionnalise la sélection, améliore le suivi et réduit les traitements informels.

---

### 4.4. Module inscriptions

Une candidature acceptée peut devenir une inscription officielle.

Le module inscriptions permet :

- de générer une inscription unique à partir d’une candidature validée ;
- d’attribuer une référence sécurisée ;
- de gérer un statut d’avancement ;
- de suivre le montant dû et le montant payé ;
- de produire des accès sécurisés au dossier ;
- d’archiver proprement les dossiers.

**Intérêt pour la DG :** ce module crée une continuité formelle entre recrutement et intégration effective de l’étudiant.

---

### 4.5. Module paiements

Ce module structure le suivi financier des inscriptions.

Il prend en charge :

- les paiements en espèces ;
- les paiements de type mobile money ;
- les virements bancaires ;
- les validations de paiement ;
- les reçus ;
- les références de transaction ;
- les sessions d’encaissement sécurisées ;
- la traçabilité des agents de paiement.

Le système prévoit également une logique importante : **la création automatique de l’étudiant après le premier paiement validé**, ce qui évite les décalages entre financier et scolarité.

**Intérêt pour la DG :** cela améliore la rigueur de gestion et la lisibilité du cycle financier.

---

### 4.6. Module étudiants

Ce module prend le relais après l’inscription et le premier paiement validé.

Il permet de créer un profil étudiant officiel avec notamment :

- un compte utilisateur ;
- un matricule ;
- un lien direct avec son inscription ;
- un statut d’activité.

**Intérêt pour la DG :** l’établissement peut passer d’une logique de dossier à une logique de population étudiante structurée.

---

### 4.7. Module actualités, blog et communication

Le site comprend plusieurs leviers de communication :

#### Actualités
Pour publier les nouveautés de l’établissement : événements, annonces, résultats, partenariats, rentrée, communiqués.

#### Blog
Pour diffuser des contenus plus éditoriaux : analyses, conseils, articles pédagogiques, décryptages, vie académique, orientation.

#### Événements et galerie
Pour valoriser la vie de l’établissement par l’image et la preuve sociale.

#### Témoignages et partenaires
Pour renforcer la confiance, la crédibilité et la preuve de valeur.

**Intérêt pour la DG :** ce bloc est essentiel pour la réputation, l’engagement et le recrutement.

---

### 4.8. Module communauté

La plateforme intègre également une dimension communautaire avec :

- espaces de discussion ;
- questions/réponses ;
- profils publics ;
- classements ;
- badges ;
- notifications ;
- abonnements à des domaines.

Cette dimension favorise :

- l’engagement ;
- la rétention ;
- l’animation de communauté ;
- la valorisation des apprenants et membres actifs.

**Intérêt pour la DG :** c’est un levier de différenciation fort par rapport à un site institutionnel classique.

---

### 4.9. Module conformité légale

Le projet intègre désormais un système de gestion des pages légales, notamment :

- les **mentions légales** ;
- la **politique de confidentialité** ;
- les **conditions d’utilisation**.

Ces contenus peuvent être pilotés depuis l’administration, avec logique de publication et historique de version.

**Intérêt pour la DG :** cela renforce la crédibilité institutionnelle et la conformité réglementaire.

## 5. Parcours utilisateur de bout en bout

Voici le parcours fonctionnel central du projet.

### Étape 1 — Découverte
Le visiteur découvre l’école via :

- la page d’accueil ;
- la page à propos ;
- les formations ;
- les actualités ;
- les contenus blog ;
- les moteurs de recherche.

### Étape 2 — Intérêt
Le visiteur consulte une formation précise et trouve :

- la description du programme ;
- les compétences ;
- les débouchés ;
- les frais ;
- les documents requis ;
- les témoignages ;
- le bouton de candidature.

### Étape 3 — Candidature
Le candidat remplit son dossier et le soumet.

### Étape 4 — Analyse interne
L’équipe admissions traite le dossier et peut :

- le mettre en revue ;
- demander un complément ;
- l’accepter ;
- l’accepter sous réserve ;
- le refuser.

### Étape 5 — Inscription
Si la candidature est acceptée, une inscription peut être créée.

### Étape 6 — Paiement
Le service financier ou l’agent concerné enregistre et valide le paiement.

### Étape 7 — Création étudiant
Après le premier paiement validé, le profil étudiant officiel peut être créé automatiquement.

### Étape 8 — Suivi institutionnel
La direction, les responsables et les agents suivent ensuite les indicateurs et les opérations depuis leurs espaces dédiés.

## 6. Gouvernance du système : qui fait quoi ?

L’un des grands intérêts du projet est la séparation claire des responsabilités.

### 6.1. Superadmin

Le superadmin est le cockpit central de l’établissement.

Il permet de gérer :

- les formations ;
- les cycles, filières, diplômes et frais ;
- les candidatures ;
- les inscriptions ;
- les paiements ;
- les étudiants ;
- les actualités, articles et événements ;
- les pages institutionnelles ;
- les partenaires et témoignages ;
- les annexes/campus ;
- les messages ;
- les utilisateurs ;
- les paramètres globaux.

**Rôle stratégique :** c’est l’espace de gouvernance, de contrôle et de configuration.

---

### 6.2. Dashboards agents

Le projet prévoit des dashboards dédiés aux personnes qui travaillent au quotidien sur des missions précises.

#### Dashboard admissions
Pour les responsables des dossiers :

- suivi des candidatures ;
- revue des documents ;
- changement de statut ;
- génération d’inscription.

#### Dashboard finance
Pour les agents de paiement :

- recherche d’inscriptions ;
- création/validation des paiements ;
- gestion des sessions de caisse ;
- consultation des détails financiers.

#### Dashboard direction / exécutif
Pour le pilotage global :

- lecture consolidée ;
- statistiques ;
- suivi transversal.

**Point important de gouvernance :** les utilisateurs de dashboards agents ne doivent pas accéder au cockpit superadmin, sauf profil expressément autorisé. Cette séparation est essentielle pour la sécurité et l’organisation.

---

### 6.3. Gestion des rôles et groupes

Le système prévoit des groupes métiers structurés, notamment :

- responsables admissions ;
- agents finance ;
- direction générale.

Cela permet de :

- donner les bonnes autorisations aux bonnes personnes ;
- limiter les erreurs ;
- sécuriser les accès ;
- améliorer la responsabilité de chaque acteur.

**Intérêt pour la DG :** le système devient gouvernable à mesure que l’équipe grandit.

## 11. Sécurité et contrôle

Le projet a été pensé avec plusieurs logiques de sécurité et de contrôle métier :

- comptes utilisateurs ;
- séparation des rôles ;
- permissions par groupe ;
- références uniques pour les inscriptions ;
- accès privés aux dossiers ;
- traçabilité des statuts ;
- agents de paiement identifiés ;
- validation métier sur plusieurs étapes clés.

### Ce que cela apporte

- moins de confusion entre les services ;
- meilleure traçabilité ;
- réduction des manipulations non autorisées ;
- meilleure qualité des données.

### Point d’attention avant production

Certains contrôles techniques de production doivent être **validés ou renforcés avant la bascule finale**, par exemple :

- politique stricte des mots de passe ;
- secret key définitive de production ;
- configuration d’hébergement sécurisée ;
- politique SSL/HTTPS ;
- sauvegardes ;
- revue des accès administratifs ;
- validation du domaine et des emails réels ;
- monitoring et journalisation.

Autrement dit, le socle applicatif est là, mais la mise en production doit s’accompagner d’une **discipline d’exploitation**.

## 16. Risques ou points d’attention à surveiller

Aucun projet numérique ne doit partir en production sans une vigilance sur les sujets suivants :

- qualité finale des contenus textuels ;
- qualité et poids des images ;
- cohérence des données de démonstration ;
- vérification complète des filtres et recherches ;
- contrôles d’accès par rôle ;
- stabilité des parcours critiques ;
- vérification des emails et notifications ;
- rigueur de sauvegarde et de monitoring ;
- capacité de l’équipe à utiliser correctement les dashboards.

Ces points ne remettent pas en cause la valeur du projet ; ils font partie des bonnes pratiques normales avant une mise en ligne officielle.

### Phase 3 — Montée en puissance
- élargir l’usage aux différents services ;

- intégrer de nouveaux flux si nécessaire (reporting, exports avancés, CRM, etc.).
